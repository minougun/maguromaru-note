## 時系列（要約）

1. **ラベル削除・点線の色**  
   - SVG の `<text>` / ピル / リーダー線を削除。  
   - 未記録の破線を黒系に変更する試み。

2. **「ラベルが残る／点線が黒くならない」問題**  
   - 原因: **`public/zukan-tuna-map.webp` にラベルやガイドが焼き込まれていた**。React 側でラベルを消しても画像上の文字は残る。  
   - 対応案: 背景を **`public/tuna-map-base.svg`**（ラベルなし）に差し替え、オーバーレイだけで表現。

3. **ユーザー要望で旧 WebP に戻す**  
   - 再び `zukan-tuna-map.webp` を使う方向に戻した。

4. **Gemini 生成 PNG への差し替え**  
   - ユーザー指定の PNG を `public/zukan-tuna-map.png` として配置（約 4.4MB）。  
   - **本番で `/zukan-tuna-map.png` が 404** になる事象あり（同じ環境で他の `public` 静的ファイルは 200）。原因は Vercel 側の取り込み要因と推測。

5. **Next バンドルに画像を載せる**  
   - `src/assets/zukan-tuna-map.png` を `import` し、`/_next/static/media/...` として配信。本番でも 200 を確認。

6. **軽量化（WebP）**  
   - `sharp` で幅 1365・品質 84 の WebP に変換（約 46KB）。  
   - `src/assets/zukan-tuna-map.webp` を import。  
   - `scripts/optimize-zukan-tuna-map.mjs` と `npm run assets:zukan-map` を追加（後述のロールバックで削除済みの経緯あり）。

7. **名称ラベルの再表示**  
   - ピル＋リーダー線＋`<text>` を復活。

8. **参考画像（714×429）に合わせた座標**  
   - `sx` / `sy` でスケール。腹を 3 楕円に分割するなどの変更を試した（`belly-chutoro-rear` など）。

9. **「ラベルが見えない」再発**  
   - **SVG の描画順**: 後から描いた大きな楕円が、**先に描いた領域のラベルを上から被せる**可能性。  
   - 対応: 楕円グループとラベルグループの **二段描画**、`globals.css` の `.map-label-*` と `.map-wrap` の `overflow` 調整などを試した。

10. **ユーザー要望: ラベルがあった頃にロールバック**  
    - **`dee98d9`（`bf40336` の直前＝ラベル削除コミットの一つ前）相当**に `TunaMap.tsx` を戻した。  
    - 複数部位カードのリード文も当時の文言に復帰。  
    - コミット例: `a30452c`（`revert(zukan): TunaMap をラベル削除直前（dee98d9 相当）に戻す`）。

11. **魚イラストも当時どおりに戻す（依頼 → ユーザーがキャンセル可）**  
    - 依頼内容: `git` から `dee98d9` の `public/zukan-tuna-map.webp`（約 42KB）を復元し、`TunaMap` を `publicPath("/zukan-tuna-map.webp")` に。`src/assets` の WebP・`sharp`・最適化スクリプトを外す、など。

## 技術メモ（トラブルシュート用）

- **`public/` の大きな PNG が本番 404**  
  - 同じプロジェクトの別静的ファイルは 200、という状況で起きた。  
  - **回避策の一例**: `import` による **`/_next/static/media/...` バンドル**。

- **ラベルはコードにあるのに見えない**  
  - 画像に焼き込みがないか確認。  
  - **後続の `<ellipse>` が前の領域の `<text>` を覆っていないか**（描画順）を疑う。

- **デプロイ後も古い**  
  - `immutable` なメディア URL のキャッシュ、ブラウザの JS バンドルキャッシュ。強制再読み込みやシークレットで確認。

- **GitHub Pages / basePath**  
  - 静的ファイルは **`publicPath()`** で prefix を付けるパターンが既にある（`TunaMap` で利用中の経緯あり）。
