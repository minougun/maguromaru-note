## デプロイ・確認のしかた

1. **推奨**: `git push origin master` 後、Vercel ダッシュボードで **Git 連携デプロイ**の成功を確認（本リポジトリの運用ポリシー）。
2. **`npx vercel --prod` / CLI**: **1日あたりのデプロイ API 上限**（例: 100 回/日）に当たることがある。CLI が弾かれた場合は **`git push` で Git 連携ビルド**を頼る（プロジェクト設定次第で Production に乗る）。
3. 本番で **強制再読み込み** またはシークレットウィンドウ。
4. 図鑑 `/zukan` で Network を見る:
   - マグロ画像が **`/_next/static/media/...webp`** であること（`public` 直下の `zukan-tuna-map.webp` は使っていない）。
5. 文言・レア度は **クライアントバンドル**に含まれるため、**古い JS を掴んでいる**と表示が変わらない。ハードリロードで切り分け。
6. チュートリアルは **コード上のモック**のため、静的画像 URL の確認は不要（変更はデプロイ後の実機でスライドを目視）。
